Risk in a Box is a suite of web based tools aimed to help understand risk to natural disasters and aid decision making for building resilience. The applications of Risk in a Box use GeoNode at their core to store and share data. The Risiko application builds on GeoNode to model impact of different hazard events on population or infrastructure. And the TsuDAT application is the first hazard modeling application in the suite It is Open Source Software, and we encourage you to build new applications using its components and the resources it provides. What is Risk in a Box made of?

Risk in a Box is built using several open source projects, each with its own community. If you are interested in contributing new features, we encourage you to do so by contributing to one of the projects on which it is built:

  • Risiko - Impact calculation engine and simple to use calculator.
  • TsuDAT - Tsunami modelation tool.
  • GeoNode - Spatial data infrastructure that integrates social and cartographic tools
  • GDAL/OGR - Translator library for raster and vector geospatial data formats
  • SciPy/Numpy - Open-source library for mathematics, science, and engineering

Who started Risk in a Box?

The first version of Risk in a Box was written in Jakarta, Indonesia in 2011. The following is the list of initial parters that contributed to the development:

  • AIFDR - The Australia Indonesia Facility for Disaster Reduction, supported by the Australian and Indonesian governments.
  • GFDRR - The Global Facility For Disaster Recovery and Reduction, The World Bank
  • BPPT - Badan Pengkajian Dan Penerapan Teknologi (Agency for the Assessment and Application of Technology)
